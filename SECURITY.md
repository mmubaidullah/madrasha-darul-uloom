## নিরাপত্তা দুর্বলতা রিপোর্ট করা

আমরা আমাদের সফটওয়্যারের নিরাপত্তাকে গুরুত্ব সহকারে নিই। যদি আপনি একটি নিরাপত্তা দুর্বলতা আবিষ্কার করেন, অনুগ্রহ করে দায়িত্বশীলভাবে এটি প্রকাশ করুন।

### রিপোর্টিং প্রক্রিয়া

**অনুগ্রহ করে পাবলিক GitHub ইস্যুতে নিরাপত্তা দুর্বলতা রিপোর্ট করবেন না।**

পরিবর্তে, অনুগ্রহ করে একটি ইমেইল পাঠান:
- **ইমেইল**: security@yourmadrasha.com
- **বিষয়**: Security Vulnerability Report - Madrasha Management System

### রিপোর্টে অন্তর্ভুক্ত করুন

আপনার রিপোর্টে নিম্নলিখিত তথ্য অন্তর্ভুক্ত করুন:

1. **দুর্বলতার বর্ণনা**: সমস্যাটি কী এবং এটি কীভাবে কাজ করে
2. **প্রভাব**: এই দুর্বলতা কী ক্ষতি করতে পারে
3. **পুনরুৎপাদনের ধাপ**: কীভাবে দুর্বলতাটি পুনরুৎপাদন করা যায়
4. **প্রমাণ**: স্ক্রিনশট, লগ, বা অন্যান্য প্রমাণ (যদি প্রযোজ্য হয়)
5. **প্রস্তাবিত সমাধান**: যদি আপনার কোনো সমাধানের ধারণা থাকে

### আমাদের প্রতিশ্রুতি

আমরা নিম্নলিখিত প্রতিশ্রুতি দিচ্ছি:

- **স্বীকৃতি**: আমরা ২৪ ঘন্টার মধ্যে আপনার রিপোর্ট স্বীকার করব
- **প্রাথমিক মূল্যায়ন**: ৭২ ঘন্টার মধ্যে প্রাথমিক মূল্যায়ন প্রদান করব
- **নিয়মিত আপডেট**: তদন্তের সময় নিয়মিত আপডেট প্রদান করব
- **ক্রেডিট**: আপনি চাইলে আমরা আপনাকে আবিষ্কারক হিসেবে ক্রেডিট দেব

### দায়িত্বশীল প্রকাশ

আমরা দায়িত্বশীল প্রকাশ নীতি অনুসরণ করি:

1. **গোপনীয়তা**: আমরা সমস্যা সমাধান না হওয়া পর্যন্ত এটি গোপন রাখব
2. **সমন্বিত প্রকাশ**: আমরা আপনার সাথে প্রকাশের সময় সমন্বয় করব
3. **ক্রেডিট**: আপনি চাইলে আমরা আপনাকে আবিষ্কারক হিসেবে উল্লেখ করব

## নিরাপত্তা সর্বোত্তম অনুশীলন

### ব্যবহারকারীদের জন্য

1. **শক্তিশালী পাসওয়ার্ড**: জটিল এবং অনন্য পাসওয়ার্ড ব্যবহার করুন
2. **নিয়মিত আপডেট**: সফটওয়্যার সর্বদা আপডেট রাখুন
3. **HTTPS**: সর্বদা HTTPS সংযোগ ব্যবহার করুন
4. **লগ আউট**: ব্যবহার শেষে লগ আউট করুন
5. **সন্দেহজনক কার্যকলাপ**: কোনো সন্দেহজনক কার্যকলাপ দেখলে রিপোর্ট করুন

### ডেভেলপারদের জন্য

1. **Input Validation**: সব ইনপুট যাচাই করুন
2. **SQL Injection**: Parameterized queries ব্যবহার করুন
3. **XSS Prevention**: Output encoding করুন
4. **Authentication**: শক্তিশালী authentication ব্যবহার করুন
5. **Authorization**: Proper access control বাস্তবায়ন করুন
6. **HTTPS**: সর্বদা HTTPS ব্যবহার করুন
7. **Dependencies**: নিয়মিত dependencies আপডেট করুন
